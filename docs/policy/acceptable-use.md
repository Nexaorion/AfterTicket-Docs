# AfterTicket 可接受使用政策

## 1. 目的与范围

本政策用于保护用户、AfterTicket Team、第三方平台、票务参与者和网络基础设施。

本政策适用于 AfterTicket Client、Control、Dashboard、官方 Master、更新服务、API、WebSocket、节点、推送功能和其他由 AfterTicket Team 提供或控制的服务。

AfterTicket Team 是位于新加坡、由自然人组成的未注册协作团队，不是注册成立的公司、法人或其他独立法律实体。

无论某项功能是否在代码、界面或配置中可用，均不表示团队授权您将其用于违反法律、第三方规则或他人权利的目的。

## 2. 可接受的使用

在遵守法律、第三方规则、活动规则和本政策的前提下，您可以：

- 为本人或已明确授权的人员配置合法购票任务；
- 使用本人所有或已获明确授权的 Bilibili 账号、Cookie、购票人和节点；
- 在合理频率下查询项目、库存和订单状态；
- 在受控环境中管理自己的节点和任务；
- 经团队事先书面授权，进行安全研究、商业部署或其他特殊用途。

## 3. 必须遵守的基本要求

您必须：

- 遵守所在地法律、新加坡法律和相关第三方平台规则；
- 仅处理您有权处理的账号、Cookie、Token、购票人和身份资料；
- 尊重 Bilibili、活动方、票务方和其他用户的公平访问机会；
- 保持合理请求频率，并遵守票务平台风控限制；
- 保护节点、Master、Dashboard、Cookie、Node Token、订单 Token 和支付信息；
- 立即停止可能造成损害、未经授权访问或异常请求压力的任务；
- 配合合理的安全、滥用和权利调查。

## 4. 禁止的票务和交易行为

您不得使用 AfterTicket：

- 从事倒票、加价转售、黄牛服务、囤票或有组织地排挤普通购票者；
- 购买法律、平台规则或活动规则禁止购买的票；
- 使用虚假、盗用、伪造或未经授权的身份和购票人资料；
- 代表他人下单但未获得其明确授权；
- 欺诈、拒付、洗钱、套现或规避付款义务；
- 绕过实名、限购、资格、地域、年龄或其他购票限制；
- 干扰库存、排队、抽签或订单分配机制的公平性；
- 提供付费代抢、节点出租、账号池、Cookie 交易或其他未经团队书面授权的商业服务。

## 5. 禁止的请求滥用与风控规避

您不得：

- 以可能造成过载、拒绝服务或异常压力的频率请求 Bilibili、AfterTicket 或任何第三方；
- 无依据地降低 `order_interval`、`stock_check_interval` 或其他节流配置；
- 删除、绕过或故意无视票务平台风险控制和退避机制；
- 使用多账号、多节点、代理或其他方式规避第三方限制；
- 通过 AfterTicket 进行压力测试、容量攻击、扫描或探测，除非获得目标所有者明确书面授权；
- 伪造来源、响应、库存、订单或遥测数据；
- 利用自动化功能操纵第三方服务或损害其他用户。

AfterTicket 对验证码、Token、风险控制响应或相关参数的技术支持，不授权您违反第三方规则或规避合法访问控制。

## 6. 禁止的账号、凭证与个人数据行为

您不得：

- 访问、使用、保存、出售、分享或传播未经授权的账号、Cookie、Session、Token 或支付信息；
- 提交、查询或使用未经相关个人授权的姓名、手机号、证件号码或购票人资料；
- 公开发布配置、日志、Cookie、Node Token、订单 Token、支付链接或二维码；
- 使用 AfterTicket 跟踪、骚扰、歧视、冒充或伤害他人；
- 收集超出合法目的所需范围的个人数据；
- 规避访问控制或尝试访问其他用户、节点、任务、订单或账号数据；
- 使用泄露、购买、窃取或以其他不正当方式取得的凭证。

## 7. 禁止的节点、网络与远程控制行为

您不得：

- 将不属于您或未经授权的设备注册为节点；
- 向不属于您或未经授权的节点发送任务、重启、更新或其他命令；
- 使用 `target_ip`、自定义 Master、代理、Host 重写或其他配置访问内网、云元数据服务、受限地址或非预期目标；
- 使用自定义更新地址分发恶意、被篡改或未经授权的软件；
- 使用 `push_config`、自定义推送服务器或成功后命令执行恶意代码、提权、持久化、数据窃取或破坏行为；
- 诱导他人运行节点、安装更新或提交 Token、Cookie 和日志；
- 干扰 WebSocket、心跳、任务队列、遥测、日志或更新机制。

## 8. 禁止的安全与完整性破坏

除非适用法律、明确许可证或团队书面授权允许，您不得：

- 绕过身份认证、权限控制、Node Token、会话或租户隔离；
- 利用、传播或隐瞒安全漏洞以造成损害；
- 上传、传播或执行恶意软件、勒索软件、后门或破坏性代码；
- 修改、伪造或移除更新包校验、版权声明、权利管理信息或来源标识；
- 反编译、反向工程或规避技术保护措施；
- 冒充 AfterTicket Team、官方服务或团队成员；
- 使用 AfterTicket 名称、域名、标识或界面实施钓鱼、诈骗或混淆。

安全研究人员应先通过本政策末尾的联系方式取得授权，并遵守约定的范围、时间、数据处理和披露要求。

## 9. 禁止的违法、侵权与有害内容

您不得使用 AfterTicket 创建、上传、发送、存储或传播：

- 侵犯版权、商标、商业秘密、隐私、个人数据或其他权利的内容；
- 欺诈、诽谤、骚扰、威胁、仇恨、暴力或违法内容；
- 用于犯罪、规避执法或破坏证据的内容；
- 违反出口管制、制裁或其他适用限制的内容；
- 误导他人认为您获得 Bilibili、活动方或 AfterTicket Team 背书的内容。

## 10. 商业使用限制

除非团队事先书面授权，您不得：

- 销售、出租、转授权或代运营 AfterTicket；
- 以 AfterTicket 提供付费代抢、商业节点、账号池、Cookie 或 Token 服务；
- 将官方服务用于商业转售、白标服务或大规模商业自动化；
- 使用 AfterTicket 品牌招揽客户、募集资金或作出商业承诺；
- 将团队或贡献者描述为您的合作方、雇员、代理或担保方。

本条不影响适用开源许可证明确授予的权利，但您仍须遵守法律、第三方规则和本政策。

## 11. 监测与执行

在适用法律允许的范围内，我们可能使用账号、节点、任务、订单、错误、日志、遥测和投诉信息调查违反本政策的行为。

根据行为性质和风险，我们可以：

- 警告并要求整改；
- 限制功能、请求或连接；
- 暂停、封禁或删除账号；
- 撤销 Session、Node Token 或节点连接；
- 停止任务、禁用节点或拒绝更新；
- 删除、隔离或限制相关内容；
- 保存证据并依法配合权利人、平台、执法或监管机关；
- 采取法律允许的其他措施。

我们会根据严重程度、影响、主观故意、历史行为和配合程度采取合理措施，但没有义务在每种情况下先行警告。

## 12. 报告滥用与安全问题

- 滥用报告：`abuse@afterticket.org`
- 安全问题：`security@afterticket.org`
- 一般联系：`contact@afterticket.org`

报告应尽量包含：

- 涉及的账号、节点、任务、URL 或时间；
- 行为描述和可能影响；
- 可复现步骤或证据；
- 您的联系方式。

请勿在首次报告中发送完整 Cookie、Session Token、Node Token、身份证件号码、支付二维码或与问题无关的个人数据。

## 13. 政策变更

我们可能更新本政策，并公布新的版本和生效日期。您在更新生效后继续使用 AfterTicket，即表示接受更新后的政策。
